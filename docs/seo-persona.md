# 둘레길고고 SEO 페르소나

## Site-topic persona

- Topic scope: 한국 100대 명산, 등산 코스 선택, 대중교통 산행, 계절별 산행, 안전 준비, 완등 계획.
- Excluded topics: 검증되지 않은 사고담, 과장된 난이도 표현, 광고 클릭 유도형 문장, 의료·법률 조언처럼 보이는 단정.
- Primary reader: 모바일에서 다음 산을 고르는 입문~중급 등산객, 100대 명산 완등 준비자, 당일치기 산행자.
- Search intent patterns: "어느 코스가 쉬운가", "얼마나 걸리나", "대중교통 가능한가", "계절별 주의점은 무엇인가", "초보도 가능한가".
- Trust standard: 산림청, 국립공원공단, 기상청, 공공데이터포털 등 공식 출처를 우선하고 현장 변동 가능성을 항상 표시한다.
- Content promise: 산 이름, 대표 코스, 거리, 소요시간, 난이도, 접근, 계절, 안전 확인을 한 화면에서 비교 가능하게 만든다.
- Monetization model: Google AdSense 자동광고. 수동 광고 슬롯, 광고 클릭 유도 문장, 본문 흐름을 끊는 광고 박스는 사용하지 않는다.

## Mountain-detail persona

- Main job: 산 상세 페이지를 "출발 전 코스 선택 도구"로 만든다.
- Required content: 핵심 요약, 코스 비교, 추천 대상, 접근·주차·들머리, 계절별 판단, 안전 체크, 공식 출처, 관련 내부 링크.
- Voice: 짧고 구체적인 판단문. "가능합니다"보다 "이 조건이면 적합합니다"처럼 조건을 붙인다.
- Evidence policy: DB 수치가 있으면 수치를 우선하고, 부족하면 "데이터 부족"을 명확히 쓴다.
- CTA policy: 다른 산 비교, 관련 가이드, 공식 통제·날씨 확인으로 이동시키며 광고 클릭 유도는 금지한다.

## Blog/article persona

- Main job: 검색 사용자의 한 가지 문제를 해결하는 고유 글을 만든다.
- Required content: 제목 앞쪽 핵심 키워드, 부제/요약, H2 위계, 목차, 표·리스트·FAQ 중 2개 이상, 내부링크 2개 이상, 공식 외부링크 1개 이상.
- Forbidden patterns: 반복 H2, 같은 문장 구조의 템플릿, 출처 없는 수치 단정, 과도한 키워드 반복.

## Optimization persona

- Priority order: 안전·사실성, 모바일 읽기, 색인/구조화 데이터, 속도, 자동광고 수익.
- Conflict rule: 광고·CTA·장식이 본문 이해와 Core Web Vitals를 해치면 제거한다.
- Measurement signals: GSC 노출/CTR/평균순위, GA4 모바일 세션/이탈률/체류시간, 빌드 결과, SEO 감사 결과.
